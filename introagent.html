The engine03 / 12

An LLM is a function.

Text in. Distribution over the next token out. Sample, append, repeat. No state. No hands. No time.

The agent04 / 12

Wrap it in a loop. Give it tools.

Same model. Now it has time. Each beat: observe, think, act, append. Until it's done.

agent ≈ LLM + loop + tools

Tools05 / 12

Tools are the hands.

The model emits a structured request. The runtime runs the actual function. The result re-enters the conversation. Anything you can wrap, it can use.

The persistence problem07 / 12

Then it forgets.

A memory you live in08 / 12

Memory as a place.

A knowledge graph the agent walks through and edits. Nodes are concepts. Edges, multichannel, are how they relate. It doesn't query its memory. It moves in it.

Dreamed nightly. Pruned. Provenanced. The agent grows its own map.
